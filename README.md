# Timeweb Домены — панель регистратора (прототип)

Кликабельный референс-прототип панели **domains.timeweb.ru** для передачи разработчикам.
Показывает интерфейс и **пути пользователя** по модели «код-верификации» через ЕСИА.

> Это не production. Данные — моки, бэкенда нет. Дизайн нейтральный и будет
> переработан командой дизайна. Цель — зафиксировать экраны, состояния и навигацию.

## Запуск

```bash
npm install
npm run dev      # http://localhost:5180
npm run build    # сборка в dist/
npm run preview  # предпросмотр сборки
```

Адаптив: один интерфейс перестраивается под ширину (mobile ↔ desktop), брейкпоинт `760px`.
Текущий путь видно в плашке «ПРОТОТИП» сверху.

## Карта путей пользователя (URL = шаг сценария)

```
/login                 Вход — только через Госуслуги (ЕСИА)
   └─ /auth/gosuslugi   Переход на Госуслуги, получение подтверждённых данных
        └─ /home        Главный экран: сводка + 2 опции
             ├─ /domains            «Мои домены» — агрегат со всех реселлеров + фильтр
             │     └─ /domains/:name  Деталь домена + действия
             └─ /code               Генерация временного кода верификации (TTL 30 мин)
```

Гард авторизации: `/home`, `/domains`, `/code` доступны только после входа,
иначе редирект на `/login` (см. `src/App.jsx`).

## Обработанные юзеркейсы (со стороны панели)

| # | Юзеркейс | Где |
|---|----------|-----|
| 1 | Вход только через Госуслуги, без логина/пароля | `Login` |
| 2 | Переход на ГУ и наследование подтверждённых данных | `GosRedirect` |
| 3 | Сводка: число доменов и реселлеров | `Home` |
| 4 | Две ключевые опции: «Мои домены» / «Сгенерировать код» | `Home` |
| 5 | Агрегированный список доменов со всех реселлеров + внешние (RU-CENTER) | `Domains` |
| 6 | Фильтр по реселлеру | `Domains` |
| 7 | Статусы домена: верифицирован / истекает / у регистратора | `StatusBadge` |
| 8 | Пустое состояние списка (нет доменов у реселлера) | `Domains` |
| 9 | Деталь домена: реселлер, срок, администратор, размещение | `DomainDetail` |
| 10 | Действия по домену: ЛК реселлера / продлить / код | `DomainDetail` |
| 11 | Генерация кода `TWD-XXXX-XXXX` + копирование | `GenerateCode` |
| 12 | Обратный отсчёт TTL и предупреждение при <5 мин | `GenerateCode` |
| 13 | **Истечение кода** → состояние «протух» + перевыпуск (новый кейс) | `GenerateCode` |
| 14 | Выход из сессии | `Layout` |

## Структура

```
src/
  main.jsx            точка входа, провайдеры, роутер
  App.jsx             роуты + гард авторизации + плашка прототипа
  store.jsx           Context-стор (сессия, пользователь, домены) — заменить на реальные API
  styles.css          нейтральные адаптивные стили
  data/mock.js        мок-данные: пользователь, домены, реселлеры, код, TTL
  components/         icons, Layout, StatusBadge
  pages/              Login, GosRedirect, Home, Domains, DomainDetail, GenerateCode
```

## Заметки для разработчиков

- **Авторизация** мокается в `store.jsx` (`login/logout`). Реальная — OAuth-флоу ЕСИА.
- **Данные доменов** в `data/mock.js`. В проде — агрегация по ЕСИА-id через API реселлеров.
- **Clipboard** (`navigator.clipboard`) требует https/localhost — в прод-сборке учесть.
- **SPA-роутинг**: при деплое нужен fallback всех путей на `index.html`
  (иначе прямой переход на `/domains` отдаст 404 на статике).
- Десктоп — это **тот же** интерфейс с адаптивом, отдельной навигации нет (по согласованию).
